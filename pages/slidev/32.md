在继续讲解之前，我们先来看看一些常见错误。一旦我们开始动手练习，就很可能会遇到它们。

标签不匹配是非常常见的编辑错误。标签是一个 html 语言中的术语。它必须成对出现。常见的标记有<div>, p等等。在 slidev 中，内置组件和第三方组件也往往是标签，需要成对。另外 slidev 还有少时内置语法，也需要成对。

在这个例子当中，标签 el 有结束，但没有开始。所以，它的提示是 invalid end tag。并且我们看它提示了源码行是第6行。把这个行号加上 frontmatter 的行数，就能对应上我们的源文件了。

[click]

这是缺少结束标签的例子。第12行有一个 el 标签，但是它没有结尾标签。

[click]
很多情况会比这两种复杂。比如，也许标签已经配对了，但是格式不对，不正确的缩进、缺少回车，也会导致语法解析器找不到匹配的标签。

在右边就显示了这样一个例子。

这段文本中，template 标签是一个 slidev 内置标签。它本来是一一对应的。但是，由于结束的 template 标签与上一行文本之间，缺少回车符，导致 template 这个标签也被当成了文本。

这种情况，只在 markdown以及基于 markdown 的 slidev 中出现。在 markdown 中，markdown 语法与 html 语法可以混写。那么要如何区分哪一些片段是 markdown 文本，哪一些是 html 标签呢？一般要求，在 html 标签之后，开始新的 markdown 文本之前，需要插入空行；同样，在 markdown 文本之后，开始新的标签之前，也要插入一个空行，否则，它们就会被都当成 markdown 文本。

注意由三个 backtits 引起的代码段也是 markdown 的一部分，它也要与前面的 html 标签之间，额外插入一个空行。

[click]

不正确地缩进也会造成问题。在 markdown 中，缩进的文本一般被当成代码，此时，它里面的任何符号都不会被解析，只会当成文本看。所以，如果某个标签藏在缩进里，那实际上就少了这个标签。它被当成普通文本了。

在右边的例子中，结尾的 template 标签虽然也换行了，但是，它与前面的列表文本一上起都加了缩进，所以，也会导致解析错误。

[click]
有时候我们感觉写对了，但会出现右下这样的格式不正确的情况。这就是 markdown 与标签之间，缺少空行导致的。在第一行与第二行之间，插入一个空行，就正确了。

[click]

一旦出现标签不对等语法错误，此时浏览器与服务器之间的连接就会断开。此时即使修复了语法错误，屏幕也不会像之前那样自动更新。此时，我们要回到启动演示的命令行，输入字母 r 来重启动，然后再刷一下浏览器。

[click]

如果我们使用多页文档来组织演示文稿，一开始我们可能在主控文档里，只加入了几个页面，后来再加入了新的页面，会发现新页面不显示。

此时需要重启服务器。主控文档只读了一次。

[click]
