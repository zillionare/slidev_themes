  <div class="ml--10">

  1. 任何出现在\<\>中的内容都是标签，包括\<h1\>~\<h6\>, \<div\> \<p\>, \<img\> 等
  2. 标签必须闭合（成对）。比如\<div\>配对的标签\</div\>。闭合的标签就成为一个合法的元素。
  3. 元素可以嵌套，也可以包含普通文本，或者不包含任何子元素。
  4. 对没有子元素的标签，也可以写作\<tag/\>。特别是，\<img\>元素这么写很常见。
  5. 元素可以有属性。最常见的属性是 class, style 等。
  6. \<div\>是特殊元素，适合作为容器，它本身没有样式和属性。
  </div>

因为 slidev 本身是一种网页开发，所以，尽管有 AI辅助，我们还是要懂一点网页的基本知识。

这个表格是 markdown 与 html 元素的对比。markdown 是一种精减的语法，所以，它的标记就那没 html 那么多。主要是一到六级标题，普通文本，分隔符，图片和表格等。

另外一个不同点就是，markdown 的元素没有属性，而 html 元素至少都有 class, style 两个属性，称为样式，也就是这个元素显示出来的大小，颜色、有无边框、位置等等。这些都是视觉效果，与内容的语义没有任何关系。所以，markdown 文本基本上只保留了内容的语义。在呈现给最终用户时，再临时加上样式，就可以成为很漂亮的网页文本。

但是，要成为演示文稿，纯粹的 markdown文本语法肯定是不够的。比如，我们要决定某一段文本呈现某种动画效果，这就需要特别加上一些标记才行。

这样 slidev 就使用了一些 html 的标记，再加上由此拓展出来的组件和addon，就能构建复杂的、有动画效果的演示文稿了。

[click]

网页的是由一系列元素构成的，元素的定义依赖于标签。这个表格列举了标签和元素最重要的一些特性。记住标签、样式、属性这些概念，我们基本上就能对 slidev 的文档理解一个大概。

[click]
